# Beiguo Market Research Skill

`beiguo-market-research` 是一个用于市场调研、产品调研、用户研究和商业机会判断的 Codex Skill。

它的目标不是写一份泛泛的行业报告，而是把真实市场里的「人、货、钱、路、规则、历史变化」拆开，再落回到用户自己的产品、服务、用户群体、渠道、定价和能力约束上，给出可以行动的产品判断。

## 适合场景

- 市场调研、行业调研、城市/门店/渠道/品类调研
- 产品机会判断、用户需求验证、MVP 验证设计
- 竞品与替代方案分析
- 用户访谈提纲、田野调研计划、报告大纲生成
- 商业模式、定价、渠道、获客路径和交付风险判断
- 把已有调研案例沉淀成可复用的方法论

## 核心方法

这个 Skill 强制调研先回到具体市场，不直接从观点出发：

- 从一个典型样本深挖，而不是泛泛覆盖
- 找内部知情人，而不是只问方便回答的人
- 记录具体品类、价格、销量、毛利、费用、渠道和交易规则
- 区分用户、买方、付款人、影响者、审批者和失败风险承担者
- 追问真实替代方案，而不是只问「你需不需要」
- 用证据等级标注结论可信度
- 最后必须输出产品决策：做、不做、缩小人群、改 offer、改价格、换渠道、先验证，或等待更多证据

## 安装

把 Skill 目录复制到 Codex 的 skills 目录：

```bash
git clone https://github.com/mufengDiamond/beiguo-skill.git
cp -R beiguo-skill/beiguo-market-research ~/.codex/skills/
```

如果你使用的是自定义 `CODEX_HOME`：

```bash
cp -R beiguo-skill/beiguo-market-research "$CODEX_HOME/skills/"
```

## 使用方式

在 Codex 中直接提到这个 Skill：

```text
Use $beiguo-market-research 帮我调研 AI 编程教育这个市场，并判断我这个产品该不该做。
```

也可以这样问：

```text
用 beiguo-market-research 帮我设计一份线下门店调研计划。
```

```text
帮我做一个产品机会判断：目标用户是中小企业老板，产品是 AI 销售培训服务。
```

## 输出内容

根据任务不同，Skill 会输出：

- 调研计划
- 用户访谈提纲
- 田野观察清单
- 市场结构图
- 品类和交易经济模型
- 用户与利益相关者结构
- 替代方案和购买链路分析
- 产品机会评分卡
- 定价和渠道判断
- MVP 或下一步验证实验
- 最终战略建议和风险清单

## 证据等级

重要判断会按证据强度标注：

- `A`：真实付款、订单、合同、留存、使用记录或现场交易
- `B`：多角色交叉验证，或一个角色加可观察证据
- `C`：单一访谈口述，可信但未验证
- `D`：公开报告、媒体、平台数据或行业 benchmark
- `E`：推断或假设，只适合用于规划下一步验证

## 设计原则

好的调研不是把行业讲得很大，而是把一个具体决策变清楚。

这个 Skill 默认追问：

- 用户是谁？
- 他现在怎么解决？
- 谁真的付钱？
- 为什么现在方案不够好？
- 他愿意为更好方案付多少钱？
- 你能不能触达他？
- 你能不能交付？
- 什么证据说明应该继续？

最终目标是把市场事实翻译成产品行动。
